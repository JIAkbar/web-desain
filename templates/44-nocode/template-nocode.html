FAQ

Pertanyaan yang sering ditanya

Apa bedanya GridBase dengan Excel atau Google Sheets? +
Excel dan Google Sheets adalah aplikasi pengolah angka yang dipaksakan jadi database. GridBase adalah database sesungguhnya yang tampilannya seperti spreadsheet. Bedanya: GridBase punya tipe data yang ketat (tidak ada salah tipe), relasi antar tabel native, multiple view (Kanban, Calendar, Gallery), permission granular, dan API bawaan — semua tanpa satu baris kode.
Bisakah saya import data dari Excel atau Google Sheets yang sudah ada? +
Ya, sangat mudah. GridBase mendukung import dari file Excel (.xlsx), CSV, dan Google Sheets langsung via URL. Saat import, GridBase otomatis mendeteksi tipe data setiap kolom dan menyarankan field type yang tepat. Proses import 1.000 baris data hanya butuh kurang dari 30 detik.
Berapa batas jumlah record dan data yang bisa disimpan? +
Plan Gratis: 1.000 record per base, attachment 1 GB total. Plan Pro: 50.000 record per base, attachment 50 GB. Plan Business: 500.000 record per base, attachment 1 TB. Tidak ada batasan jumlah field per tabel. Jika data kamu lebih besar, hubungi kami untuk paket Enterprise dengan kapasitas custom.
Apakah GridBase bisa diakses offline? +
GridBase adalah aplikasi berbasis cloud, jadi membutuhkan koneksi internet untuk sinkronisasi real-time. Namun kami sedang mengembangkan mode offline untuk aplikasi mobile (estimasi rilis Q3 2026) yang memungkinkan kamu membaca dan mengedit data, dengan sinkronisasi otomatis saat koneksi tersedia kembali.
Apakah ada REST API dan bagaimana cara menggunakannya? +
Ya. Setiap base di GridBase otomatis punya REST API endpoint. Kamu bisa GET, POST, PATCH, dan DELETE record via API key yang bisa dibuat di Settings. Dokumentasi API tersedia di docs.gridbase.id dengan contoh curl, JavaScript (fetch), Python, dan PHP. Rate limit: 100 req/hari (Gratis), unlimited (Pro & Business).
