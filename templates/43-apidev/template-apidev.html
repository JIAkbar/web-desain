⚡
Rate Limiting

Konfigurasi per-key, per-IP, atau per-endpoint. Sliding window, fixed window, atau token bucket — pilih sesuai kebutuhan.

🔐
Auth Multi-Layer

OAuth 2.0, JWT validation, API Key, dan mTLS dalam satu gateway. Ganti strategi tanpa ubah kode backend.

📊
Analytics Real-time

Request per menit, error rate, latency distribution, dan top consumers — semua live tanpa konfigurasi tambahan.

🔍
Log & Debug

Full request/response log dengan replay. Filter by status code, endpoint, atau consumer. Integrasikan ke Datadog atau Loki.

🔀
API Versioning

Kelola v1, v2, v3 secara paralel. Traffic splitting, deprecation warning otomatis, dan sunset policy per versi.

🚀
Caching Layer

Edge cache dengan TTL per-route. Cache bypass via header. Kompatibel dengan Redis custom atau managed cache FluxAPI.

Siap dalam 3 Baris Kode

SDK tersedia untuk semua bahasa utama. Atau langsung pakai REST — tidak ada lock-in.

curl -X GET https://api.fluxapi.io/v2/users \
  -H "Authorization: Bearer {API_KEY}" \
  -H "Content-Type: application/json" \
  -H "X-FluxAPI-Version: 2024-06"
// npm install @fluxapi/sdk
const flux = require('@fluxapi/sdk');

const client = new flux.Client({
  apiKey: process.env.FLUX_KEY,
  version: '2024-06'
});

const users = await client.users.list({ limit: 20 });
console.log(users.data); // [{ id, name, email, ... }]
# pip install fluxapi

Pertanyaan Teknis

Ya. Plan Enterprise mendukung deployment via Docker image atau Helm chart ke cluster Kubernetes kamu sendiri. Self-hosted instance tetap sync ke dashboard cloud FluxAPI untuk analytics dan konfigurasi, atau bisa fully air-gapped jika diperlukan.
FluxAPI mendukung rate limiting per API Key, per IP, per user ID dari JWT claim, atau kombinasinya. Kamu bisa set limit berbeda untuk tier consumer yang berbeda via konsep "Consumer Groups". Semua konfigurasi via dashboard atau Terraform provider.
Overhead p50 FluxAPI adalah 3–8ms, p99 di bawah 20ms — diukur dari edge PoP terdekat. Kami punya 14 edge location termasuk Jakarta dan Singapura. Untuk request yang hit cache, overhead bisa turun ke <1ms.
Kami menyediakan migration script untuk Kong (via Deck config) dan import wizard untuk AWS API Gateway JSON export. Untuk Nginx, tersedia panduan manual konversi rule. Traffic dapat di-shadow route dulu — request tetap ke gateway lama, FluxAPI hanya mirror untuk comparison sebelum full cutover.
Secara default, FluxAPI hanya log metadata (URL, method, status, latency, headers). Request/response body logging bisa diaktifkan per-route dengan redaction rules untuk field sensitif (password, token, kartu kredit). Data log disimpan di region yang kamu pilih dan bisa di-export ke S3 atau GCS.
