Cara Kerja

3 Langkah, Workflow Berjalan

Tidak butuh pengalaman coding. Siapapun bisa membangun workflow otomasi yang powerful.

1

Pilih Trigger

Tentukan kapan workflow berjalan — form submission, webhook, jadwal waktu, email masuk, atau event dari app lain.

trigger: form_submission
source: "Formulir Website"
filter: has_email == true
2

Tambah AI Actions

Seret dan lepas action nodes — AI Agent, kondisi bercabang, delay, transformer data, atau kirim ke integrasi.

action: ai_classify
model: "gpt-4o-mini"
prompt: "Klasifikasikan lead..."
3

Aktifkan Flow

Klik Aktifkan — workflow langsung berjalan. Monitor real-time, lihat log setiap eksekusi, debug jika ada masalah.

runs_today: 2,847
success_rate: 99.8%

FAQ

Pertanyaan yang Sering Ditanyakan

Apakah data saya aman saat diproses AI Agent? ›
Data Anda diproses dalam lingkungan terisolasi dan terenkripsi AES-256. Kami tidak pernah menggunakan data pengguna untuk melatih model AI pihak ketiga. Semua prompt yang dikirim ke model AI diproses secara ephemeral — tidak disimpan oleh provider model. AutoFlux juga memiliki opsi on-premise untuk enterprise yang memerlukan data sovereignty penuh.
Bisakah saya membuat trigger dan kondisi yang sangat spesifik? ›
Ya. AutoFlux mendukung trigger dari webhook custom, cron expression (jadwal fleksibel), event dari 200+ integrasi, atau bahkan polling API eksternal setiap N menit. Kondisi percabangan mendukung logika boolean kompleks, perbandingan string/angka, regex matching, dan ekspresi JavaScript sederhana untuk logika yang tidak tertangani UI visual.
Apakah ada batasan kecepatan (rate limit) eksekusi workflow? ›
Tier Free dibatasi 10 eksekusi/menit. Tier Starter 100 eksekusi/menit. Tier Business 1.000 eksekusi/menit dengan burst tolerance hingga 5.000. Jika Anda memiliki kebutuhan di atas itu (misalnya flash sale e-commerce), hubungi tim kami untuk paket Enterprise dengan rate limit custom dan dedicated infrastructure.
Bagaimana cara debug workflow yang gagal? ›
Setiap eksekusi dicatat di Execution Log lengkap dengan input dan output masing-masing node, waktu eksekusi, dan pesan error detail. Anda bisa klik eksekusi mana saja dan jalankan ulang dengan data aslinya (replay run). Untuk mode debug, aktifkan "Step Mode" — workflow berjalan node per node sehingga Anda bisa inspect data di setiap titik. AI Assistant kami juga bisa menganalisis error log dan menyarankan perbaikan.
Berapa SLA uptime AutoFlux dan apa yang terjadi jika platform down? ›
Tier Business mendapatkan SLA 99.9% uptime (downtime maksimal ~8.7 jam/tahun). Jika terjadi gangguan, event-driven workflow yang belum dieksekusi akan masuk ke queue dan otomatis dijalankan saat platform pulih — tidak ada data yang hilang. Kami menyediakan status page real-time di status.autoflux.id dan notifikasi insiden aktif via email/Slack.
